Написать веб-приложение без аутентификации, например, гостевая книга, на любом языке и развернуть его в Yandex Cloud с использованием serverless-технологий.

Причем у него:
Статический фронт, доступный из интернета. Реализован с помощью Object Storage, Serverless Functions или Serverless Containers. В UI приложения должно быть видно версию фронта.
Есть реплики бэкенда, реализованные либо с помощью Serverless Functions, либо с помощью Serverless Containers. В UI приложения должно быть видно версию бэкенда.
Каждый раз попадаем на разные реплики бэкенда за счет скрытия бэкенда за API Gateway.
Данные приложения, например, все сообщения гостевой книги, сохраняют в Serverless YDB.
Есть публичный доступ из интернета по HTTPS. Достаточно доступа через бесплатные служебные домены Yandex Cloud.

В репозиторий веб-приложения добавить скрипты для PowerShell или Bash с использованием Yandex CLI для автоматизации развертывания приложения:
Обновление Serverless Containers
Обновление Serverless Functions
Создание схемы в YandexDB (как вариант скрипт может запускать приложение в особом режиме, в котором приложение не запускается, а создается схема)

Предоставить Ивану Домашних (digisturm@yandex.ru) достаточный доступ к облаку, чтобы проверить инфраструктуру, а также работоспособность скриптов. Вероятно, что потребуется дать доступ большему числу проверяющих. В этом случае список будет отправлен в чат курса в Telegram.

Предоставить ссылку на веб-приложение.


* Вместо гостевой книги можно использовать ваше приложение с проектного практикума, но оно должно удовлетворять всем описанным в задании требованиям

** Бонусное задание:
Приобрести свой домен (это за ваш счет, поэтому это бонусное задание)
Зарегистрировать в DNS-серверах Yandex Cloud свой домен
Настроить DNS так, чтобы ваш домен или его поддомен указывал на API Gateway вашего веб-приложения
Подключить к нему сертификат через Let’s Encrypt, чтобы работал https

*** Основная идея этого задания — научиться разворачивать приложения таким образом, чтобы их не пришлось удалять в следующем семестре. Достигается это за счет того, что при использовании serverless-технологий надо платить только за использование, а еще достаточно большое количество запросов можно сделать вообще бесплатно. Если при выполнении задания разворачивать приложения из проектного практикума, то можно пополнить свое портфолио ссылками свои на всегда доступные (и бесплатные) приложения
